Enumeration Type Documentation

enum GRAPHIMPL

Definition of backing implementation (array-based, linked-lists, hashtable/unique adjacency lists)

Type of underlying graph implementation used. Each type implies certain graph characteristics (fixed vs. dynamic) and drives the graphops function implementations

Enumerator
GENERIC 

An unspecified, growable implementation. It is assumed that the user does not care about the underlying details, merely that the graph behaves in a manner that supports the most general behavior (growable, dynamic, etc).

ARRAY 

Array-backed graph data.

Nodes, edges, and capacities are maintained in adjacency array form. The graph dimensions (counts, edges, membership) are expected to be fixed at initialization.

LINKED 

Linked-list backed graph data.

Underlying graph structures are dynamically allocated and adjustable at runtime.

HASHED 

Hashtable-based implementation of adjacency lists. Management of hashtables and id-lookups is handled by the graphops implementations. Graph structure is dynamic and adjustable at runtime.

enum GRAPHTYPE

Basic graph type.

Type of graph being implemented. DIRECTED vs. UNDIRECTED gives appropriate hints to the graph operations that will be used, both to access the data and to minimize memory usage.

Enumerator
UNDIRECTED 

Undirected graph (edge u->v == edge v->u, always).

Structure implementations will make all efforts to ensure no duplicates, and only one representative edge will be used.

DIRECTED 

Directed graph (edge u->v != edge v->u).

Structure implementations will maintain separate values for each direction. More memory, but (at times) faster or more explicit path operations.
